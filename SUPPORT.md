# Support

Workflow Skill Router is an open-source local developer tool. Support is community-based and provided on a best-effort basis.

## Where to Ask

- Use [GitHub Discussions](https://github.com/eric861129/Workflow-skill-router/discussions) for installation questions, routing policy design, SKILL-only usage, and general troubleshooting.
- Use a focused issue form for a reproducible routing failure, Plugin/MCP runtime bug, or evaluation case.
- Read the [Plugin installation guide](site/src/content/docs/guides/install-plugin.md), [SKILL-only guide](site/src/content/docs/guides/install-skill.md), and [MCP tool reference](site/src/content/docs/reference/mcp-tools.mdx) first.

## Security and Sensitive Data

Security reports must use GitHub private vulnerability reporting as described in [SECURITY.md](SECURITY.md). Do not put vulnerabilities, credentials, private prompts, SQLite state, internal paths, or unsanitized traces in Issues or Discussions.

## What Maintainers Need

Include exact versions, installation mode, operating system, a minimal reproduction, expected behavior, actual behavior, and sanitized logs. Evaluation questions should also identify the adapter kind, attempt count, manifest digests, and evidence status.

Maintainers may close issues that cannot be reproduced, belong to a third-party product, expose sensitive information, or request support for an unsupported V1 version.
