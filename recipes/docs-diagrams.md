# Recipe: Docs And Diagrams

Use this route when the deliverable is reader-facing documentation, architecture explanation, or diagrams.

## Good Route

```text
Route: Documentation / Architecture guide > C4 or process diagram > Engineering audience
Use SKILL: documentation-writer, diagramming, architecture-designer
Reason: documentation-writer structures the page; diagramming creates the visual model; architecture-designer checks system boundaries.
```

## File Artifact Route

```text
Route: Documentation / File artifact > Presentation or document > Executive audience
Use SKILL: file-format-tool, documentation-writer
Reason: file-format-tool preserves rendering fidelity; documentation-writer improves narrative structure.
```

## Avoid

- Choosing a generic docs skill when the task needs a real Word, spreadsheet, slide, or PDF artifact.
- Loading every diagramming skill for a small README update.
- Writing docs without checking the repository's existing information architecture.
