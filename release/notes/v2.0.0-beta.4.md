# Workflow Skill Router v2.0.0-beta.4 — local release candidate

This source tree prepares `v2.0.0-beta.4`; it is not yet published. Stable `latest` remains V1.3.1, and the latest published `latest-v2` installation snapshot remains `v2.0.0-beta.3` until the reviewed release workflow completes.

Its checked-in `release_lifecycle` is `prepared-local-candidate`. The trusted default-branch workflow requires `reviewed-attested-publishable` bound to the reviewed `release_source_revision`, so entering `CREATE_V2_RELEASE` alone fails before preflight, tag creation, or publication.

## Why V2

As a Codex environment accumulates Skills, Plugins, and MCP tools, the hard problem becomes choosing the smallest capability set for the current work boundary. V2 makes that choice explicit, preserves user-selected Skills, reroutes each Phase or Goal Work Item, and fails closed when runtime authority is unavailable.

## Choose your install mode

- Plugin + MCP provides the bundled local control plane, durable routing plans, scoped consent transitions, and public runtime readiness. The locally prepared asset name is `workflow-skill-router-plugin-v2.0.0-beta.4.zip`.
- Skill-only preserves the routing and Explicit Skill Lock instructions without durable MCP state. The locally prepared asset name is `workflow-skill-router-skill-v2.0.0-beta.4.zip`.

Until this candidate is published, install the immutable `v2.0.0-beta.3` release instead of assuming these beta.4 asset names exist on GitHub.

## What changed

- Contract 2.3.0 measures implicit structural classification for Single, Phased, and Managed Goal work without `requested_work_mode`.
- Profile explain miss behavior is represented through stable public-safe reason codes.
- Evaluation adds `envelope_source_match`, `classification_reason_match`, `local_authority_boundary_match`, `profile_explain_match`, and `unnecessary_consent_violation`.
- New hard violations detect goal-bound local mutation, local output that claims activation, and semantic candidates persisted as authoritative routes.
- Attempt identity seals the nonce, tool inventory, instruction digest, public case payload digest, model/reference-driver version, and an irreversible scorer-side digest of the private oracle and scoring policy.
- Every Contract 2.3 turn must provide bounded evidence using one public non-oracle reason-code vocabulary; missing, invalid, or undeclared evidence blocks attestation.

## Case-set continuity

The full suite remains 13 cases. The existing structural cases now carry the Contract 2.3.0 evidence oracle, and `profile-explain-miss` replaces `evaluation-manual-required`. The beta smoke remains six cases with one two-turn scoped-consent case, preserving the planned 36 attempts and 42 model turns. Historical Contract 2.2.0 reports keep their original case and instruction digests and are never rescored.

## Evidence boundary

The deterministic reference-driver validates protocol composition, sealed binding, scoring, and artifact reproducibility. Reference-driver evidence does not prove real-model behavior, real Skill activation, reduced token use, or task Outcome. A fresh model run still requires explicit quota authorization, a frozen source/case/instruction/model digest, sanitized-evidence review, zero hard violations, and maintainer attestation.

The required `evaluation_evidence` output contains only allowlisted stable codes and booleans. Scoring expectations never enter the execution driver, public case payload, or sanitized case diagnostics. A transcript sealed under a different scoring-spec digest is rejected instead of being silently resumed or rescored.
