# Workflow Skill Router v2.0.0-beta.3

## Why V2

Workflow Skill Router plans the smallest verifiable Skill route before Codex starts work. It separates installed metadata from real runtime authority, preserves explicitly named Skills, reroutes at Phase boundaries, supports user-owned Skill Trees, and binds long-running work to Host-owned Goal state without claiming permissions it does not have.

Beta.3 is a security and supply-chain hardening release. It does not change the Router contract or expand runtime authority.

## What changed in beta.3

- Removed the privileged cross-workflow checkout that OpenSSF classified as a Critical dangerous workflow.
- Bound Pages deployment to the trusted `main` revision only after deterministic artifacts, core contracts, and site visual gates pass in the same workflow.
- Moved Lighthouse's scoped Sentry dependency to the patched OpenTelemetry 2.x line, closing `GHSA-8988-4f7v-96qf` without a forced downgrade or broad override.
- Added offline regression coverage for the safe OpenTelemetry lockfile boundary and the trusted Pages deployment invariant.

### Skill-only

Download `workflow-skill-router-skill-v2.0.0-beta.3.zip` and extract its inner `workflow-skill-router/` directory into the Codex Skills directory. Skill-only preserves task-shape routing, Explicit Skill Lock, reporting, and advisory Profile interpretation. Loading fixed local Profile files requires Host filesystem access; it does not provide durable consent, compare-and-swap, drift enforcement, or `hybrid-full` conformance.

## Evidence boundary

The reviewed 36 attempts and 42 model turns belong to `v2.0.0-beta.1` under evaluation contract 2.2.0. They remain valid historical evidence for that runtime candidate. Beta.3 changes CI deployment and documentation-site development dependencies, not routing behavior, so it does not relabel beta.1 evidence or claim a new model evaluation.

Personal Routing Profile claims remain supported by deterministic contract, integration, security, migration, package, and documentation evidence. A new real-model evaluation still requires separate quota authorization, sanitized evidence review, zero hard violations, and a new `maintainer-attestation`.
