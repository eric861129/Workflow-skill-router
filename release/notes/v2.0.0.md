# Workflow Skill Router v2.0.0 — formal GA candidate

`v2.0.0` is a formal GA candidate and is not yet released. The
checked-in `release_lifecycle` is `prepared-local-candidate`; it cannot create a tag,
publish assets, or move a release channel. A later metadata-only promotion
must bind `release_source_revision` to the reviewed frozen GA candidate,
record fresh final model evidence and a maintainer-attestation, and set
`reviewed-attested-publishable` before `Release V2` may run.
`CREATE_V2_RELEASE` cannot bypass this lifecycle and evidence guard.

Stable recovery remains pinned to immutable `v1.3.1`. The latest published V2
snapshot is historical `v2.0.0-beta.3`; do not use an installation command for
`v2.0.0` until the trusted workflow has created that immutable tag.

## Why V2

Workflow Skill Router is a pre-execution Skill-selection layer. It chooses a
small, explainable route before work begins, preserves explicit Skill choices,
and reports runtime capability boundaries. It does not grant permissions,
approve actions, replace sandboxing, activate Skills, or provide production
orchestration or Host authority.

## Choose your install mode

- **Plugin + MCP** is the full candidate surface: durable Router-owned plans,
  scoped consent transitions, and runtime capability reporting. It exposes
  four always local-ready operations and three conditional-local operations
  only for a validated Router-owned work graph.
- **Skill-only** keeps the routing and Explicit Skill Lock instructions when
  Plugin/MCP is unavailable. It is advisory and has no durable MCP state or
  Host authority.

For both modes, Native Goal transitions, verified activation, deployment, and
production actions remain outside the Router. A conditional-local gate is only
an advisory local result; it is not completion or approval.

## Candidate scope

- The **Router-owned Local Work Loop** provides `get_next_work`,
  `record_work_event`, and `evaluate_gate` only as bounded
  `conditional-local` operations. The complete claim is **4 always local-ready + 3 Router-owned conditional-local**, never `7/12 local-ready`.
- The **Host Integration Kit** contains a capability manifest, reference
  adapter, and conformance suite. It is not verified production Host authority
  and does not permit a `hybrid-full` claim.
- The frozen Pilot protocol is preparation only. No real Pilot task has been
  executed or counted.

## Evidence boundary

Tier 0 Contract fixtures and the reference-driver prove deterministic contract
composition. The reference-driver does not prove real-model behavior.
Historical beta evidence is not a final GA qualification. No fresh final GA model qualification, completed
Pilot result, verified Host deployment, or maintainer-attestation is claimed by
this candidate. Final qualification normally requires an authorized 36 attempts /
42 model turns run bound to the frozen source revision, sanitized evidence review,
zero hard violations, and trusted maintainer attestation. A reviewed **Delta
Qualification** evidence bridge is permitted only for a precommitted monotonic
safety repair that is exactly bound to an existing full report, its former hard
violation, an allowlisted source diff, and a machine-checked postcondition. It
adds narrow confirmation to the parent full qualification; it never replaces
that qualification or the zero-hard-violations and attestation requirements.
