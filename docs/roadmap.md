# Roadmap

Workflow Skill Router is intentionally small: it should stay a routing layer, not become a large meta skill. The roadmap focuses on better public examples, stronger validation, and easier adoption.

## v1.2

- Strict CI gates for public safety, duplicate skill ids, and exact routing expectations.
- Metadata-rich sample skills for scanner demos and generated skill trees.
- Clean installable template package without non-essential per-skill README files.
- More obvious README and website proof sections for first-time visitors.

## v1.3

- Larger routing benchmark with more boundary and anti-over-routing scenarios.
- Community route gallery for reusable public-safe workflow examples.
- Agent-agnostic adapter notes for tools beyond Codex.
- Better guidance for generating predictions from a router without leaking private context.

## Future

- Marketplace-style skill catalog viewer for browsing domains, stages, triggers, and exclusions.
- Automatic prediction generation guide for repeatable route evaluation.
- More shareable demos showing fuzzy requests becoming reviewable routing output.
