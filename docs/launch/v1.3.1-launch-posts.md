## Dev.to

```markdown
---
title: "Workflow Skill Router: a small routing layer for multi-skill AI agents"
published: false
tags: ai, opensource, github, productivity
---

Modern AI coding agents can have many skills, tools, and workflows available at once. That power is useful, but it also creates a practical problem: fuzzy requests can trigger too many skills, which makes the result harder to review.

Workflow Skill Router is an open-source starter for adding a small routing step before execution begins.

Instead of:

```text
frontend, ui, browser, playwright, qa, design-system, github, docs, deployment
```

the agent chooses a bounded route:

```text
Route: Frontend / Debugging > Browser reproduction > Single-page app
Use SKILL: vue-expert, systematic-debugging, playwright
Reason: vue-expert handles component behavior; systematic-debugging keeps the investigation causal; playwright captures the regression.
```

v1.3.1 focuses on making the project safer and easier to share:

- public-safe validation for docs, examples, downloads, and site assets
- Markdown/MDX local link checks in CI
- MP4/WebM demo with poster and GIF fallback
- route gallery and benchmark metrics from the v1.3 line
- dependency governance for development-only audit advisories

Links:

- Repository: https://github.com/eric861129/Workflow-skill-router
- Docs: https://huangchiyu.com/Workflow-skill-router/
- Demo: https://huangchiyu.com/Workflow-skill-router/showcase/
- Release: https://github.com/eric861129/Workflow-skill-router/releases/tag/v1.3.1

The project is intentionally small: a Blank Router starter, a reference template, validation scripts, and public-safe examples. If you work with skill-based AI agents, I would love feedback on the routing contract and route case format.
```
