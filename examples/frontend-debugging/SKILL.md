# Frontend Debugging Router

Use this router when a frontend issue could involve rendering, state, routing, API data, browser session state, or test automation.

## Workflow

1. Decide whether the first evidence should come from an interactive browser, a scripted test, source inspection, or API logs.
2. Select one Primary skill and up to three Supporting skills.
3. Explain the route before reproducing or fixing the issue.
