# Company Platform Router

Use this router for anonymized enterprise platform work. It demonstrates how to keep a public router reusable while leaving private repository names and deployment details out of the shared example.

## Workflow

1. Classify the request by company platform area, work stage, and technical domain.
2. Prefer platform-specific skills when the task targets platform architecture, API contracts, UI boundaries, deployment, RBAC, or incidents.
3. Add generic engineering skills only when they cover a distinct job.
4. Keep the route to one Primary skill plus up to three Supporting skills.
