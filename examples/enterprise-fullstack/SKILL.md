# Enterprise Fullstack Router

Use this router for company product engineering tasks that span more than one discipline.

## Workflow

1. Classify the task by product area, work stage, and technical domain.
2. Prefer a company platform skill when the task targets company-specific architecture.
3. Add connector skills only when live remote state is needed.
4. Keep the selected set to one Primary skill plus up to three Supporting skills.

## Output

```text
Route: task nature > work stage > technical domain
Use SKILL: primary-skill, supporting-skill, supporting-skill
Reason: one short sentence per SKILL
```
