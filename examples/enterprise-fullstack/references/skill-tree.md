# Workflow Skill Tree

## Product / Requirements / Planning

- Product / Scope / Cross-team feature: Primary: `requirements-clarity`; Supporting: `product-architecture`, `test-planning`
- Planning / Implementation / Multi-service change: Primary: `implementation-planning`; Supporting: `platform-architecture`, `release-checklist`

## Backend / API / Data

- Backend / API contract / Customer-facing service: Primary: `api-designer`; Supporting: `backend-developer`, `database-design`, `test-planning`
- Backend / Data model / Migration or indexing: Primary: `database-design`; Supporting: `backend-developer`, `database-optimizer`
- Backend / Integration / External provider: Primary: `integration-architect`; Supporting: `api-designer`, `security-review`, `documentation-writer`

## Frontend / Portal / Admin

- Frontend / Customer portal / Feature implementation: Primary: `frontend-builder`; Supporting: `ui-design`, `api-client-generation`, `browser`
- Frontend / Internal admin / Dense operational workflow: Primary: `internal-ui-design`; Supporting: `frontend-builder`, `browser`, `accessibility-review`
- Frontend / Debug / Runtime behavior: Primary: `frontend-debugging`; Supporting: `browser`, `systematic-debugging`, `api-debugging`

## Documentation / Release

- Documentation / Technical guide / Integration or workflow: Primary: `documentation-writer`; Supporting: `diagramming`, `api-designer`
- CI / Failure / Build or test pipeline: Primary: `ci-debugging`; Supporting: `github-connector`, `devops`
- Release / Readiness / Company platform: Primary: `release-checklist`; Supporting: `github-connector`, `ci-debugging`, `browser`
