# Template Skill Catalog Router

Use this example to understand how the downloadable template package can be organized into practical workflow routes.

Before starting a complex task:

1. Classify the request by task nature, work stage, and technical domain.
2. Select exactly one Primary skill and up to three Supporting skills.
3. Prefer narrow task skills over broad meta skills.
4. Keep connector-specific, organization-specific, or local-only rules outside this public example.

For complex tasks, respond with:

```text
Route: task nature > work stage > technical domain
Use SKILL: primary-skill, supporting-skill, supporting-skill
Reason: one short sentence per SKILL
```

For simple tasks, respond with:

```text
No extra routing needed: reason
```

Reference files:

- `references/skill-tree.md`: the template route catalog.
- `references/routing-rules.md`: selection and conflict rules.
- `references/sample-routes.md`: copyable route examples.
