# Common Engineering Skill Tree

This public example uses real skill names from a mature development workflow, but removes organization-specific private skills.

## Requirements / Planning / Task Breakdown

- Requirements / Clarify / Complex feature: Primary: `requirements-clarity`; Supporting: `superpowers:brainstorming`, `superpowers:writing-plans`
- Planning / Implementation plan / Multi-stage engineering: Primary: `executing-plans`; Supporting: `superpowers:executing-plans`, `karpathy-guidelines`
- Finish / Handoff / Branch completion: Primary: `session-handoff`; Supporting: `finishing-a-development-branch`, `superpowers:verification-before-completion`, `commit-work`

## Architecture / API / Backend

- Architecture / System design / High-level decisions: Primary: `architecture-designer`; Supporting: `c4-architecture`, `mermaid-diagrams`, `cloud-architect`
- API / Contract design / REST or GraphQL: Primary: `api-designer`; Supporting: `openapi-to-typescript`, `code-documenter`
- API / REST governance / Naming, pagination, compatibility, errors: Primary: `api-guidelines-skill`; Supporting: `api-designer`, `openapi-contract-generation-skill`
- API / OpenAPI lifecycle / Schema diff, client generation, SDK sync: Primary: `openapi-contract-generation-skill`; Supporting: `openapi-to-typescript`, `api-designer`, `build-web-apps:frontend-testing-debugging`
- Backend / C# or .NET / Implementation: Primary: `csharp-developer`; Supporting: `dotnet-core-expert`, `database-schema-designer`, `qa-test-planner`

## Database / SQL

- Database / Schema / Migration: Primary: `database-schema-designer`; Supporting: `sql-pro`, `build-web-apps:supabase-postgres-best-practices`
- Database / Performance / Slow query: Primary: `database-optimizer`; Supporting: `sql-pro`, `systematic-debugging`
- Data contract / API and DB boundary: Primary: `api-designer`; Supporting: `openapi-to-typescript`, `database-schema-designer`

## Frontend / Web / UI

- Frontend / New app or page / Vue or general web: Primary: `build-web-apps:frontend-app-builder`; Supporting: `frontend-design`, `ui-ux-pro-max`, `vue-expert`
- Frontend / Vue 3 composition / Composables and shared state: Primary: `vue-composition-patterns-skill`; Supporting: `vue-expert`, `build-web-apps:frontend-testing-debugging`
- Frontend / Public-facing portal / Customer portal or landing page: Primary: `design-taste-frontend`; Supporting: `high-end-visual-design`, `frontend-design`, `vue-expert`
- Frontend / Internal admin / Dashboard, forms, tables, permissions: Primary: `minimalist-ui`; Supporting: `frontend-design`, `vue-expert`, `browser:control-in-app-browser`
- Frontend / Redesign / Upgrade existing app without behavior drift: Primary: `redesign-existing-projects`; Supporting: `frontend-design`, `vue-expert`, `browser:control-in-app-browser`

## Frontend Debugging / Browser / Playwright

- Frontend / Debug / Browser reproduction: Primary: `build-web-apps:frontend-testing-debugging`; Supporting: `browser:control-in-app-browser`, `playwright`, `systematic-debugging`
- Frontend / Regression / Repeatable browser test: Primary: `playwright`; Supporting: `build-web-apps:frontend-testing-debugging`, `systematic-debugging`
- Chrome / Logged-in QA / Cookies, extensions, current tabs: Primary: `chrome:control-chrome`; Supporting: `browser:control-in-app-browser`, `playwright`, `systematic-debugging`
