# Common Engineering Routing

This example shows a larger, realistic engineering skill router distilled from common multi-skill development workflows.

Use it when you want examples for:

- backend, API, OpenAPI, and database routing
- frontend, Vue, design-system, browser, and Playwright routing
- docs, review, CI, DevOps, security, analytics, and connector workflows
- routing with real skill names instead of placeholder names

## Sample prompts

- `Design a new REST endpoint, generate OpenAPI types, and make sure the frontend contract can follow it.`
- `Debug a browser-only bug in an internal admin dashboard and add a regression check.`
- `Prepare a PR for a security-sensitive auth/RBAC change and verify CI.`
