# Common Engineering Router

Use this as a larger public example of a workflow skill router. It demonstrates real route shapes with concrete skill names while keeping project and organization details generic.

## Workflow

1. Classify the request as `task nature > work stage > technical domain`.
2. Read `references/skill-tree.md` when choosing between engineering domains.
3. Read `references/routing-rules.md` when skills overlap.
4. Select exactly one Primary skill and up to three Supporting skills.
5. Explain the route before starting work.

## Output Contract

```text
Route: task nature > work stage > technical domain
Use SKILL: primary-skill, supporting-skill, supporting-skill
Reason: one short sentence per SKILL
```

For simple tasks:

```text
No extra routing needed: reason
```
