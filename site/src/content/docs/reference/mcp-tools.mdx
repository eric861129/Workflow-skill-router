This page renders `site/src/data/mcp-tools.generated.json`, which is generated from the MCP server contracts. Descriptions and readiness are not duplicated by hand.

`plan_work` exposes **deterministic automatic classification** plus an **optional deterministic Profile** as two distinct sources. Planned Skill intent and actual activation remain separate: activation is `unverified` until Runtime Capability Discovery supplies evidence. Explicit Skill Lock and scoped consent still apply. The bundled local R0 core is not a semantic model, cannot activate Skills or mutate a native Codex Goal, and grants no deployment/production authority.

## Runtime profile

Current generated profile: **{toolData.runtime_profile}**

This generated profile describes the **prepared GA candidate**, which is **not included in published beta.3**. It exposes **4 always local-ready + 3 Router-owned conditional-local** tools. Conditional-local does not mean `7/12 local-ready`: it is available only for a validated Router-owned work graph with no Native Goal authority. Explicit Skill Lock and consent remain unchanged. A local gate is advisory and is not Skill activation, deployment, production approval, or Host evidence.

Native Goal boundaries remain tool-specific: `verified-host-scheduler` for next work, `verified-event-store` plus `activation-receipt-verifier` for formal progress, and `verified-evidence-store` plus `gate-authority` for gates. Missing graphs require local creation or replay through `router-owned-work-graph`; corrupt graphs return a sanitized `internal-error`. Every unavailable or unsafe branch must **fail closed**.

| Availability | Meaning |
| --- | --- |
| `local-ready` | Executable in the bundled local R0 control plane |
| `conditional-local` | Router-owned work graph only; local progress and local gate evidence remain advisory |
| `verified-host-required` | Requires verified Host authority or stores |
| `configured-adapter-required` | Requires a server-configured evaluation adapter |
